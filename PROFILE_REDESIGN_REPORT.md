## Key Features

### 🎬 Reels Endpoint (`GET /users/:id/reels`)

**Parameters**:
- `id` - User UUID or username
- `page` - Page number (default: 1)
- `page_size` - Results per page (default: 12, max: 50)

**Response**:
```json
{
  "reels": [
    {
      "id": "uuid",
      "video_url": "string",
      "likes_count": 5,
      ...
    }
  ],
  "page": 1,
  "page_size": 12
}
```

### 📍 Distance Calculation

- **Algorithm**: Haversine formula
- **Accuracy**: ±0.1m per 1000km
- **Data Source**: Redis GEO store
- **Format**: "{distance}km away" (e.g., "15.3km away")
- **Conditions**: Only shown to authenticated users viewing other profiles

### 🔒 Privacy Rules

1. **Ghost Mode**: ✅ Users in ghost mode → 403 Forbidden (except self)
2. **Block List**: ✅ Blocked users → 403 Forbidden
3. **Self Access**: ✅ Always allowed regardless of privacy settings

### 📱 Frontend UI

- **Tab Navigation**: Stories | Posts | Reels | Common
- **Grid Layout**: 3 columns (responsive)
- **Loading**: Spinner during fetch
- **Empty State**: "No reels yet" message
- **Engagement**: Likes count overlay
- **Interaction**: Hover scale animation (105%)

## API Reference

### Endpoint: GET /users/:id/reels

**Authentication**: Required  
**Rate Limiting**: Standard (inherits from general limiter)

**Query Parameters**:
```

page_size=12     (optional, default: 12, max: 50)
```

**Success Response (200)**:

**Error Responses**:
- `400 Bad Request`: Invalid user ID
- `403 Forbidden`: Ghost mode or blocked user
- `404 Not Found`: User not found
- `500 Internal Server Error`: Database error

## Known Limitations

1. **Distance Requires Location**: Distance is null if either user hasn't shared location
2. **Single Country Scale**: Haversine formula assumes sphere, ±0.2% error on Earth
3. **Cache TTL**: Profile distance updates every 10 minutes maximum
4. **Pagination Only**: No cursor-based pagination (limit/offset only)
5. **Real-time Counts**: Engagement counts may lag by seconds
